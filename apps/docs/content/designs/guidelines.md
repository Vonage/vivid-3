# Design tokens explained

Design tokens are the foundational layer of the Vivid Design System. They encode design decisions (such as colour, spacing, typography, and layout) into named, reusable values that are shared consistently across design and code.

This system introduces a layered token architecture to improve scalability, clarity, and maintainability:

- **Core tokens** → raw design values
- **Semantic tokens** → meaning-based abstractions
- **Pattern tokens** → component-level compositions
- **(Future) Component tokens** → fully encapsulated component styling contracts

This structure ensures a clear separation between what a value is, what it means, and where it is used.

## Core tokens

Core tokens represent the primitive design values of the system. They are:

- Neutral in meaning
- Not tied to any specific UI usage
- Stable building blocks for all higher-level tokens
