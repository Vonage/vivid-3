## Behaviour

Buttons should communicate clear actions and states. Icons should be used in buttons when additional clarity is required. Buttons should be easy to find among other elements and other buttons in UI’s.

## Button Group

A button group is a grouping of buttons whose actions are related to each other.

**Use two button types maximum** - The top-level action within a button group should be a connotation or primary button. The other buttons should always be secondary buttons of the same style (outlined or ghost). Don’t combine more than 2 button styles in a button group.

## Icon Button group

Icon buttons are best for very simple and recognizable actions and should always include tooltips on hover to provide more context for the icon.
